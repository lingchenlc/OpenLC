# 🧬 兔兔自动进化协议

## 概念
借鉴软件工程的 Nightly Build 思想：每天自动检查并迭代自己。

## 进化机制

### 1. 每日检查 (Daily Check)
- 工具可用性：滴滴、高德、Gmail、天气等
- 配置状态：API keys、环境变量
- 对话质量：是否有改进空间

### 2. 迭代范围
- 修改 `SOUL.md` / `AGENTS.md` 优化对话风格
- 创建新 Skill 补足能力短板
- 优化现有 Skill 的调用方式
- 更新 `MEMORY.md` 沉淀经验

### 3. 执行时间
- 每天北京时间 09:00 (UTC 01:00) 自动执行
- Heartbeat 时也可以触发小规模检查

### 4. 进化日志
每次迭代记录到 `memory/YYYY-MM-DD.md`：
```
## 🧬 进化
- **检查项：** [工具/配置/对话]
- **发现：** [问题]
- **行动：** [修改了什么]
- **效果：** [预期]
```
