## 2026-02-23 19:34 UTC

### Claude Agent SDK - 编程式构建 AI Agents

**概述:**
- 使用 Python 或 TypeScript 构建生产级 AI agents
- SDK 提供与 Claude Code 相同的工具、agent loop 和上下文管理

**核心能力:**
1. **Built-in Tools**: Read, Write, Edit, Bash, Glob, Grep, WebSearch, WebFetch
2. **Hooks**: 在 agent 生命周期关键点运行自定义代码
   - 可用于: 验证、日志、阻止或转换 agent 行为
   - 示例: 记录所有文件更改到审计日志
3. **Subagents**: 生成专业子代理处理聚焦子任务
4. **MCP**: 连接外部系统 (数据库、浏览器、API)

**Python 示例:**
```python
from claude_agent_sdk import query, ClaudeAgentOptions

async for message in query(
    prompt="Find and fix the bug in auth.py",
    options=ClaudeAgentOptions(allowed_tools=["Read", "Edit", "Bash"])
):
    print(message)
```

**自我进化启示:**
- 可用 Agent SDK 开发自定义 specialized agents
- 利用 hooks 实现自我监控和日志
- 与 MCP 结合扩展能力边界

## 2026-02-23 11:34 UTC

### Claude Code Memory & Subagents 最佳实践

**Memory 层级架构:**
- Managed policy (全局) → Project memory → User memory → Auto memory
- CLAUDE.md 支持 `@import` 语法引入其他文件
- `.claude/rules/*.md` 模块化规则，支持 `paths` 前缀过滤
- Auto memory: Claude 自动记录学习内容到 `~/.claude/projects/<project>/memory/`

**Subagents (子代理):**
- 位置: `.claude/agents/` (项目级) 或 `~/.claude/agents/` (用户级)
- 可配置: tools, model, permissionMode, maxTurns, skills, hooks
- 用途: 保留上下文、实施约束、复用配置、控制成本

**自我进化启示:**
- 使用 subagent 分离探索和执行任务
- 利用 auto memory 记录调试 insights
- 模块化 rules 保持主文件简洁
- 定期审视和修剪 memory
