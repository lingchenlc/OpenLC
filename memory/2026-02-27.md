来源: Stack-ai.com

**四种工作流架构类型:**
- **Single Agent Workflow**: 单一 Agent 拥有决策权，适合简单任务
- **Hierarchical Multi-Agent**: 层级式多 Agent，有 supervisor 控制

- 架构选择比"让模型更聪明"更重要

### 2. Observational Memory - 革命性 AI 记忆架构

来源: Mastra (Gatsby 原团队)

**核心突破:**
- **成本降低 10 倍**
- **长上下文 benchmark 超越 RAG**

**技术原理:**
- 两个后台 Agent: Observer + Reflector
- 30k tokens 触发压缩为"观察日志"
- 文本压缩 3-6x，工具输出压缩 5-40x
- 消除动态检索，完全可缓存

**性能:**
- LongMemEval (GPT-5-mini): **94.87%**
- GPT-4o: **84.23%** vs RAG 80.05%

**适用场景:** 长期运行的 agent 工作流
**不适用:** 开放性知识发现、合规性召回
